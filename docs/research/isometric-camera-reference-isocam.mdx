## Overview

IsoCam is a lightweight Blender addon (single Python file, ~200 lines) that creates isometric cameras with preset angles optimized for different use cases. It demonstrates a clean, user-friendly approach to isometric camera creation in Blender 2.8+.

### Why This Matters

This addon is popular in the Blender community for technical visualization and game development. Its simplicity and focus on practical presets provide valuable insights for our VR Production Toolkit's isometric camera implementation.

---

## Key Features

### 1. Multiple Isometric Presets

The addon provides **three distinct isometric camera presets**, each optimized for specific use cases:

#### TrueIsocam (Mathematically Correct Isometric)
**Purpose:** True isometric projection with perfect 30° side angles
**Target Users:** Technical drawings, architectural visualization, CAD-style renders

```python
# Rotation (Euler XYZ order)
Rotation X: 54.736° (0.955324 radians)
Rotation Y: 0°
Rotation Z: 45° (0.785398 radians)

# Camera Position
Location: (30.61, -30.61, 30.61)

# Projection
Type: Orthographic
Ortho Scale: 14.123
```

**Key Insight:** Uses **54.736°** tilt instead of the commonly cited 35.264°. This produces mathematically correct isometric with 30° angles on all three axes when projected to 2D.

#### GameIsocam (2D Game Isometric)
**Purpose:** Isometric for 2D games, optimized for stair rendering
**Target Users:** Game developers, pixel artists

```python
# Rotation (Euler XYZ order)
Rotation X: 60° (1.047198 radians)
Rotation Y: 0°
Rotation Z: 45° (0.785398 radians)

# Camera Position
Location: (30.61, -30.61, 25.0)

# Projection
Type: Orthographic
Ortho Scale: 14.123
```

**Key Insight:** Uses **60° tilt** which is steeper than true isometric. This is the classic "2:1 pixel ratio" isometric commonly used in strategy games and RPGs. Stairs and vertical elements render more naturally at this angle.

#### GameIsocam4to3 (Alternative Game Format)
**Purpose:** Isometric for games with 4:3 aspect ratio tiles
**Target Users:** Pixel art games, retro-style games

```python
# Rotation (Euler XYZ order)
Rotation X: 41.5° (0.724312 radians)
Rotation Y: 0°
Rotation Z: 45° (0.785398 radians)

# Camera Position
Location: (23.43, -23.43, 37.45)

# Projection
Type: Orthographic
Ortho Scale: 14.123
```

**Key Insight:** Shallower **41.5° tilt** provides more top-down view. Optimized for 64×48 pixel base tiles (common in pixel art games).

## Implementation Details

### Camera Creation Pattern

```python
# Standard pattern used by IsoCam
def create_isometric_camera(rotation_x, rotation_y, rotation_z, location, ortho_scale):
    # 1. Create camera at specific location
    bpy.ops.object.camera_add(location=location)

    # 2. Get reference to created camera
    camera = bpy.context.active_object

    # 3. Set rotation (must be done after creation)
    camera.rotation_euler = (rotation_x, rotation_y, rotation_z)

    # 4. Configure as orthographic
    camera.data.type = 'ORTHO'
    camera.data.ortho_scale = ortho_scale

    # 5. Activate immediately
    bpy.ops.view3d.object_as_camera()
```

**Note:** Rotation must be set after camera creation because `camera_add()` doesn't properly handle rotation parameter.

### Consistent Scaling Strategy

All three camera presets use the **same orthographic scale (14.123)**. This suggests:
- Standardization for predictable framing
- Designed for scenes with ~10 Blender unit base scale
- Easy to switch between presets without reframing

### Position Symmetry

Camera positions follow a pattern:
- X and Y are mirrored (positive X, negative Y)
- Z height varies by preset (higher Z = more top-down)
- Creates diagonal viewing angle (northeast perspective)

```
TrueIsocam:    (30.61, -30.61, 30.61)  ← Balanced XYZ
GameIsocam:    (30.61, -30.61, 25.00)  ← Lower Z (steeper)
GameIso4to3:   (23.43, -23.43, 37.45)  ← Higher Z (shallower)
```

## Mathematical Analysis

### True Isometric (54.736°) vs Traditional Isometric (35.264°)

**Traditional Isometric (35.264°):**
- Uses `arctan(1/√2)` = 35.264°
- X, Y, Z axes appear at 120° to each other
- Standard in CAD and technical drawings

**True Isometric (54.736°):**
- Uses `arctan(√2)` = 54.736°
- Produces exact 30° angles on all three axes in 2D projection
- More "correct" for pure isometric geometry

**Our Spec Currently Uses:** 35.264° (traditional)
**Recommendation:** Support both as separate presets

### Game Isometric Angles

**60° Tilt (GameIsocam):**
- Produces 2:1 pixel ratio (rise over run)
- One vertical pixel for every two horizontal pixels
- Industry standard for isometric games (Diablo, Civilization, etc.)
- Stairs with 1:1 depth-to-height look natural

**Why 60° Works for Games:**
```
tan(60°) = √3 ≈ 1.732
This creates the 2:1 pixel stepping when projected to screen
```

### Add Quick Create Operator

```python
class ISOMETRIC_OT_QuickCreate(Operator):
    """Quick create isometric camera with preset"""
    bl_idname = "isometric.quick_create"
    bl_label = "Quick Create"
    bl_options = {'REGISTER', 'UNDO'}

    preset: StringProperty()

    def execute(self, context):
        # Create camera with preset
        parent, camera = create_isometric_camera(
            context=context,
            projection_type=self.preset,
        )

        # Auto-activate (like IsoCam)
        context.scene.camera = camera
        bpy.ops.view3d.object_as_camera()

        # Add basic scene elements
        if context.scene.isometric_settings.show_grid:
            add_reference_grid(context)

        return {'FINISHED'}
```

## Code Examples from IsoCam

### Minimal Camera Creation (IsoCam Style)

```python
def create_game_isometric_camera():
    """Simple game isometric camera (IsoCam style)"""
    import bpy
    import math

    # Create camera
    bpy.ops.object.camera_add(location=(30.61, -30.61, 25.0))
    camera = bpy.context.active_object

    # Set rotation
    camera.rotation_euler = (math.radians(60.0), 0, math.radians(45.0))

    # Configure orthographic
    camera.data.type = 'ORTHO'
    camera.data.ortho_scale = 14.123

    # Activate
    bpy.ops.view3d.object_as_camera()

    return camera
```

### Groundplane Creation (IsoCam Style)

```python
def create_groundplane():
    """Create reference groundplane (IsoCam style)"""
    import bpy

    bpy.ops.mesh.primitive_plane_add(size=1.0, location=(0, 0, 0))
    plane = bpy.context.active_object
    plane.scale = (5.0, 5.0, 5.0)
    plane.name = "Groundplane"

    return plane
```

## Testing Recommendations

### Test: Angle Accuracy

```python
def test_true_isometric_angles():
    """Verify TRUE_ISOMETRIC uses 54.736° tilt"""
    import math

    parent, camera = create_isometric_camera(
        context=bpy.context,
        projection_type='TRUE_ISOMETRIC',
    )

    expected_tilt = math.atan(math.sqrt(2.0))  # 54.736°
    actual_tilt = camera.rotation_euler.x

    assert abs(actual_tilt - expected_tilt) < 0.001
    print("✓ TRUE_ISOMETRIC angle correct (54.736°)")


def test_game_isometric_angles():
    """Verify GAME_ISOMETRIC uses 60° tilt"""
    import math

    parent, camera = create_isometric_camera(
        context=bpy.context,
        projection_type='GAME_ISOMETRIC',
    )

    expected_tilt = math.radians(60.0)
    actual_tilt = camera.rotation_euler.x

    assert abs(actual_tilt - expected_tilt) < 0.001
    print("✓ GAME_ISOMETRIC angle correct (60°)")
```
