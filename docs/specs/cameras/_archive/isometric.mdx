## Overview

The Isometric Camera creates a true parallel projection camera with precise isometric angles for technical visualization, architectural drawings, and game art. It eliminates perspective distortion, making it perfect for technical documentation, strategy games, and pixel art.

### What is Isometric Projection?

Isometric projection is a method of visually representing three-dimensional objects in two dimensions where the three coordinate axes appear equally foreshortened. The camera views the scene at 45° horizontal rotation and 35.264° (arctan 1/√2) vertical angle, creating the classic isometric look.

**Key Benefits:**
- **No perspective distortion** - Parallel lines remain parallel
- **True measurements** - Objects maintain scale regardless of distance
- **Predictable rendering** - Consistent look across all objects
- **Industry standard** - Matches CAD and technical drawing conventions
- **Game-ready** - Perfect for isometric games and pixel art
- **Clear visualization** - Ideal for complex technical assemblies

## Workflow

### Quick Start (Single Step)

**Create Isometric Camera**
   - Click "Add Isometric Camera" in the VR Production Toolkit panel
   - Camera appears with standard isometric angles (45° pan, 35.264° tilt)
   - Positioned at origin with orthographic projection
   - Scene includes reference grid and lighting
   - Ready to render!

### Detailed Workflow

1. **Add Camera & Scene**
   - Add isometric camera with one click
   - Camera automatically positioned with correct angles

5. **Render & Export**
   - Render single frame or animation
   - Export as PNG (transparency), JPEG, or EXR
   - Optional: render with line art overlay
   - Use in technical documentation or games

## Projection Types

### Isometric (True Isometric)
**Angles:** 45° rotation, 35.264° tilt
**Scale ratio:** 1:1:1 (all axes equal)
**Use case:** Standard technical drawings, most common
**Appearance:** All three axes equally foreshortened

### Dimetric
**Angles:** 45° rotation, 30° tilt (customizable)
**Scale ratio:** 1:1:2 (two axes equal)
**Use case:** Video games, less steep than isometric
**Appearance:** Two axes equal, one different

### Trimetric
**Angles:** Custom rotation and tilt
**Scale ratio:** 1:2:3 (all axes different)
**Use case:** Specialized technical visualization
**Appearance:** All three axes differently foreshortened

### Military Projection (Plan Oblique)
**Angles:** 0° tilt (top-down), 45° rotation
**Scale ratio:** Special case
**Use case:** Maps, site plans, floor layouts
**Appearance:** Top view with depth indication

## Best Practices

### For Technical Drawings
1. **Use exact angles** - Stick to 45°/35.264° for true isometric
2. **Disable denoising** - Keep edges sharp
3. **Use flat lighting** - Avoid dramatic shadows
4. **Enable Freestyle** - Add technical line art
5. **Render high resolution** - 4K or higher for print

### For Game Art
1. **Match game projection** - Use same angles as game engine
2. **Consider dimetric** - Often better for games than true isometric
3. **Use consistent scale** - Same orthographic scale for all assets
4. **Transparent background** - Easy integration into game engine
5. **Power-of-two resolutions** - 1024, 2048, 4096 for game assets

### For Architectural Viz
1. **Add reference grid** - Shows scale and alignment
2. **Use context objects** - Add people/furniture for scale
3. **Layer multiple views** - Combine isometric with plans/elevations
4. **Label elements** - Use text objects or post-process
5. **Consistent materials** - Professional, clean material setup

### For Pixel Art
1. **Low resolution** - 512×512 or lower
2. **No anti-aliasing** - Sharp pixel edges
3. **Indexed color** - Limited color palette
4. **Consistent lighting** - Same setup for all assets
5. **Grid alignment** - Snap to pixel grid

## Tips & Tricks

### Perfect Alignment
- Use "Align to Grid" to snap camera position to world grid
- Enable "Show Grid" to visualize alignment
- Parent objects to empties for precise group positioning
- Use Blender's snap tools (Shift+S) for object placement

### Crisp Renders
- Disable denoising for technical drawings
- Use sufficient samples (256+) to avoid noise
- Enable Freestyle for clean line art
- Render at 2× size and scale down for anti-aliasing

### Consistent Asset Library
- Save camera setup as preset
- Use same orthographic scale for all assets
- Create template .blend file with camera pre-configured
- Use naming convention for consistent file organization

### Performance
- Lower samples for isometric (less complex than perspective)
- Disable unnecessary effects (DOF, motion blur)
- Use simple materials for technical work
- Optimize geometry (isometric reveals all faces)

## Troubleshooting

### Problem: Objects appear distorted
**Solution:** Ensure camera type is set to "Orthographic", not "Perspective"

### Problem: Angles don't look right
**Solution:** Verify rotation is 45° and tilt is 35.264° for true isometric

### Problem: Scale is wrong between objects
**Solution:** Check orthographic scale is consistent; objects maintain relative size in orthographic

### Problem: Edges look jagged
**Solution:** Increase render resolution or enable anti-aliasing; avoid denoising which softens edges

### Problem: Can't see entire model
**Solution:** Increase orthographic scale value to zoom out; use "Frame Selection" helper

### Problem: Lighting looks flat
**Solution:** Expected for technical drawings; add subtle rim light if desired; ensure main light is from top-left

### Problem: Transparent background not working
**Solution:** Enable "Transparent" in Film settings and render to PNG format

### Problem: Freestyle lines too thick/thin
**Solution:** Adjust Freestyle line thickness in render layer settings; typical range 1-3 pixels

## Technical Notes

### Math Behind Isometric Angles

True isometric projection uses:
- **35.264°** vertical tilt = `arctan(1/√2)` = `arctan(0.7071)`
- **45°** horizontal rotation = Equal X and Y contribution
- **Result:** All three axes (X, Y, Z) appear at 120° to each other

### Orthographic vs Perspective

**Orthographic:**
- Parallel projection rays
- No depth falloff
- Objects same size regardless of distance
- Used for technical, architectural, game art

**Perspective:**
- Converging projection rays
- Depth falloff (objects smaller with distance)
- Matches human vision
- Used for realistic rendering, photography

### Integration with Game Engines

Many game engines use isometric or dimetric projections:
- **Unity:** Orthographic camera with isometric angles
- **Unreal:** Orthographic mode with custom angles
- **Godot:** Orthographic projection settings
- **GameMaker:** Built-in isometric support

Export from Blender matches these engines when using identical angles and orthographic projection.
