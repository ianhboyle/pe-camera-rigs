## The Live Controller Workflow

### Why a Controller-Based System is Superior

Instead of creating a static camera with fixed settings, this new workflow involves adding a single **"Isometric Controller"** object. This object has a **Geometry Nodes modifier** that procedurally generates and controls the camera.

1.  **Add Controller:** User adds the rig via `Add > Camera > Isometric Controller`.
2.  **Select Controller:** The user selects the newly created controller object.
3.  **Go to Modifiers:** All camera controls are now located in the **Modifier Properties** tab.
4.  **Tweak & Animate:** The user can now change presets, adjust the zoom, and even keyframe properties directly in the modifier panel.

This is a more standard, powerful, and "Blender-native" way of working.

## What the Controller Does

Adding the "Isometric Controller" creates a rig where all logic is handled by the Geometry Nodes modifier.

1.  **Creates Controller:** An empty object named `Isometric_Controller` is created.
2.  **Attaches Modifier:** A Geometry Nodes modifier named "Isometric Camera Controller" is added to the empty.
3.  **Generates Camera:** The node group *procedurally generates* the camera object, ensuring it's always correctly parented and configured.
4.  **Links Properties:** The UI properties in the modifier panel (like `Type` and `Ortho Scale`) are directly wired into the node group to control the camera's rotation, zoom, and guide visibility.
5.  **Activates Camera:** The newly generated camera is automatically set as the active scene camera.

## Procedural Guides

The optional scene elements are now generated procedurally by the Geometry Nodes group, keeping your scene outliner clean. They can be toggled on or off at any time from the modifier panel.

*   **Show Reference Grid:** Generates a wireframe grid aligned to the world floor. Its size can be controlled by another input in the modifier.
*   **Show Axis Indicators:** Generates simple XYZ axis lines at the world origin for orientation.
